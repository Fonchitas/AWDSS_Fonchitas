CSS Frameworks

Bootstrap No Node.js

Bootstrap is kind of a classic. It was one of the first things I heard about when I started learning web design, and even though there are newer tools now, it is still useful. I like how it helps you build layouts quickly without getting too complicated. The grid system is intuitive, and the pre-built components like buttons, modals, and navbars make things really fast to set up. It is especially helpful for students or anyone who wants to get something functional online in just a few hours.

That said, a lot of websites made with Bootstrap tend to look pretty similar, especially when people stick to the default themes. For me, it is a good option for small or practical projects, like a university assignment or a client portfolio that needs to be finished fast. But for something more creative or visually unique, I would rather use tools that give me more freedom over the design. Still, I appreciate how stable and well documented Bootstrap is. It is a great safety net when you need to build something that just works.

TailwindCSS Requires Node.js

TailwindCSS is honestly amazing. I love how it gives you full control of the design directly from the HTML. You can shape everything exactly how you want. At first, the utility classes can be overwhelming because there are so many, but once you get used to the logic, it becomes really natural. I like that you do not have to jump back and forth between HTML and CSS files constantly; everything happens in one place. It feels very modular and creative at the same time.

What I really love is how Tailwind combines perfectly with other tools like Shadcn UI or Flowbite. For example, you can create something minimal and aesthetic but still functional. I have seen a lot of modern portfolios, agencies, and product websites built with it. They look clean, confident, and very adaptable to different devices. It is definitely a tool I want to keep mastering because it fits my way of working: designing by feeling, experimenting, and refining little details until it feels right.

Component Libraries

Material UI Requires Node.js

Material UI gives me a sense of order and structure. It is clean and professional, and it has that typical Google product feeling. Everything looks balanced, readable, and intuitive. I like how it takes care of accessibility and responsive behavior, which can sometimes be tricky to manage by hand. It is great for projects where clarity and consistency matter, like dashboards, apps, or client websites that follow corporate guidelines.

However, it can also feel a bit too rigid visually. The design language is very recognizable, so it is harder to make it look truly yours. Still, I appreciate how much it helps with workflow. You can combine it with other tools like Next.js or React to build complex interfaces faster. I would use it for serious, structured projects where functionality and reliability are more important than experimenting visually.

Shadcn UI Requires Node.js

Shadcn UI feels much more modern and customizable. It is built to work beautifully with TailwindCSS, which I love. The components feel lighter and more personal. You can adapt them without breaking the structure. I really like its focus on simplicity, spacing, and clean typography, which aligns a lot with my design preferences. It reminds me of the aesthetics of modern creative studios that mix clarity with personality.

Using it, I feel more in control. It lets me design components that match my own style, minimal but expressive. It is also open source and constantly evolving, which is inspiring. I would use Shadcn UI for portfolio projects or web designs where the visual language really matters, and I want to balance practicality with creativity. It is basically the type of library that lets your personal design identity shine through the code.

Static Site Generators

Jekyll Requires Ruby

Jekyll is one of those tools that helped shape how we think about static websites today. I like that it is simple, reliable, and works really well with GitHub Pages. You can write your site in Markdown and focus on the content instead of worrying about databases or hosting. It is perfect for small blogs or documentation projects, where you just want to write and publish without too many technical steps.

I do not love the Ruby setup because it feels a bit old school compared to Node.js based tools, but I respect its stability. It has been around for years, and a lot of designers still use it for personal blogs or portfolios. I could see myself using it if I wanted to create a minimal site that focuses entirely on text and storytelling, something that does not need animations or dynamic features but feels calm and intentional.

Hugo Standalone

Hugo feels like a refreshing alternative. It is super fast and does not depend on Node.js, which makes it easier to set up and maintain. I really like tools that just work straight away. You do not have to fight with dependencies or installs. It is great for portfolios, blogs, or creative journals where speed and simplicity are the priority.

I also like that it is written in Go, which makes it incredibly efficient. You can build large sites instantly. I think Hugo would be ideal for something like a personal archive of projects, photos, or writings, something lightweight but elegant. It does not have as much visual flexibility as Tailwind, but its structure and performance make it a very practical choice for clean, content driven sites.

Animation Libraries

GSAP No Node.js

GSAP is one of my favorite animation tools. I love how it makes motion feel intentional. It is not about throwing random effects, it is about bringing meaning to the interface. You can do smooth transitions, scroll based animations, or subtle movements that make a design come alive. And the best part is you can just add it with a simple script tag from a CDN, no setup needed.

I have seen GSAP used in websites for creative agencies and musicians where the animation tells part of the story. I would love to experiment with it more in my own projects, maybe for my music portfolio or visual storytelling experiments. It feels like a bridge between design and rhythm. It adds that sense of flow I always look for when I mix visual and musical ideas.

Framer Motion Requires Node.js

Framer Motion feels like the future of web motion. It is smooth, elegant, and deeply integrated with React. I like how natural its transitions look. Everything moves with purpose and precision. It is the kind of animation that does not distract you but enhances the user experience.

Even though I am still learning React, I can see how powerful this tool can be. You can animate routes, components, and even page transitions effortlessly. It gives that feeling of professional polish that you see in modern interfaces. I think I would use Framer Motion for a personal portfolio or a brand project, where I want the user to feel that flow between sections.

Templates and UI Kits

HTML5 UP No Node.js

HTML5 UP is perfect for quick, good looking setups. You can grab a template, open it in your browser, and it already looks great. There is something satisfying about how accessible it is. You do not need Node.js, Git, or any complex setup. It is ideal for personal sites, portfolios, or simple event pages where you just want to focus on the message rather than the backend.

I also like that their templates are free and open source, with beautiful design consistency. Some of them have this nostalgic early web charm but still feel clean and modern. I think it is a great way to start experimenting with layouts and understanding responsive behavior without getting lost in technical details. It is a great learning resource too.

Flowbite Requires Node.js

Flowbite feels more up to date and integrates perfectly with TailwindCSS, which makes everything consistent visually. The components like navbars, cards, dropdowns, or modals look professional and adaptable. I appreciate how customizable it is. You can start with something prebuilt and tweak it until it fits your own design system.

I could see myself using Flowbite for a portfolio or a small client project where I want a modern and clean look without spending hours coding every element from scratch. It is practical but still gives enough creative room to make something unique. It feels like the best of both worlds: pre made structure with personal freedom.

Critical Reflection (ATELIER)

My thoughts

After exploring all these tools, I have realized that I work best with the ones that give me creative freedom, like TailwindCSS and Shadcn UI. I enjoy being able to adjust everything visually and build something that feels like my own. I have always been drawn to projects that balance structure with experimentation, where you can design freely but still rely on a solid foundation.

I also understand the value of frameworks like Bootstrap or Material UI, especially when structure and usability come first. They are lifesavers when you need to deliver something fast and consistent. But personally, I am more motivated by design systems that leave room for discovery and customization. That is why tools like Tailwind and GSAP excite me. They let me mix precision with emotion.

Finding the balance between simplicity and flexibility is key for me. Sometimes the simplest tools, like HTML5 UP or Hugo, let me focus on content and storytelling. Other times, tools like Tailwind or Framer Motion give me the freedom to express a visual rhythm or personality through design. In the end, I see these tools like instruments. Each one fits a different song. Depending on the project, I can choose whether I want something more technical, more expressive, or a bit of both. That is the creative part I enjoy the most.
